一、缺口分析逻辑

步骤1：数据采集与字段映射

1

采集范围

两批合并：昨日新入库全量（status=ingested，约106K条）+ 历史库随机抽样30K（30页×1000）。排除 status=ready（= Sida AI自制视频，非真实社交内容）。

2

关键字段（曾经踩过坑）

平台：source_info.source → "yt" / "tk" / "ig"（注意：source_info.type 固定="crawler"，不能用来判断平台）

互动量：metadata.properties.likes（TK/IG无曝光数据，统一用点赞作代理指标）

步骤2：平台基准线（各自独立中位数）

不用全局中位数——YT/TK/IG 受众规模和点赞文化差异极大，混用会让 TK/IG 内容看起来永远"更好"，实际只是平台基数不同。

platform_median_yt = 1,210 点赞 （YT 所有内容的中位数） platform_median_tk = 4,679 点赞 （TK 所有内容的中位数） platform_median_ig = 5,476 点赞 （IG 所有内容的中位数）

步骤3：维度归类

对每条内容的 title 按预定义关键词树做最长匹配，归入 L1 → L2 维度。维度树共 62 个 L2，覆盖瑜伽、冥想、舞蹈、力量训练、跑步、游泳、足球、篮球、棒球、拳击、体操、综合拉伸、羽毛球、乒乓球、高尔夫、网球、健身综合、运动营养等大类。

步骤4：ROI 计算（需求强度）

per_item_roi = item_likes / platform_median -- 单条内容是平台平均水平的几倍 dim_roi = median(per_item_roi) for dim × platform -- 取中位数，抗单条爆款干扰 combined_roi = weighted_avg(dim_roi per platform, weights=content_count)

举例：K-pop舞蹈 TK 上有 5 条，中位点赞 47,446，TK 全局中位数 4,679 → ROI = 47,446 / 4,679 = 10.1×

步骤5：Gap Score 公式

gap_score = combined_roi × (1000 / sqrt(supply + 1)) -- supply = 该维度当前库内内容条数 -- +1 = 避免 supply=0 时除以0（0供给时 sqrt(1)=1，ROI 贡献完整保留） -- ×1000 = 量纲放大，让分数可读 -- sqrt = 非线性压制：供给翻4倍，分数只减半；避免线性除法的悬崖效应

逻辑：需求高+供给少 → 高分；需求高但库已很多 → 被压制；需求本来就低 → 分数也低，不值得堆。

步骤6：两类维度处理

维度类型特征处理方式
有得分维度supply ≥ 1，ROI 可计算，gap_score 有意义按 gap_score 排序，按阈值决定是否发 query（见步骤7）
零供给维度supply = 0，gap_score 无意义（ROI 无样本）单独处理，每维度直接发 20 条 query 盲补
本次分析：30个有得分维度 + 32个零供给维度，共 62 个 L2 维度全覆盖。

步骤7：缺口 → Query 转化逻辑 ⭐

① 哪些维度发 Query？— 阈值判断（非 Top N）

⚠️ 本轮修正：初始版本错误地使用了"Top 20"这个任意截止点（位置20→21 gap分数差值仅33分，无自然断层）。已修正为阈值规则：
规则纳入条件本次结果说明
有得分维度ROI ≥ 1.0×28个纳入（排除 ROI=0.54×/0.55×）需求至少达到平台平均水平
低ROI维度0.5× ≤ ROI < 1.0×2个纳入但减量（15条）信号偏弱，少发探探
极低ROIROI < 0.5×暂不发需求不足，发了也收不到好内容
零供给维度supply = 0（不论ROI）32个全发 20条/维无样本无法算ROI，盲补探索

后续改进方向：n_queries = 10 + floor(gap_score / 500) × 5，按缺口强度动态分配配额。

② 每个维度发多少条？— 三档

维度类型条数/维度依据
高缺口（gap_score ≥ 400）25条 + 15条追加 = 40条缺口最明确，值得密集覆盖
中缺口（200 ≤ gap < 400，ROI ≥ 1.0×）20条有信号，中等覆盖
低 ROI（0.5× ≤ ROI < 1.0×）15条信号偏弱，探索性发送
零供给维度20条无ROI，保守探索

③ Query 的 10 类角度扩展

每个维度的 query 沿以下角度展开，确保抓回内容覆盖不同搜索意图和难度层级：

1. 基础入门 beginner / basics / complete guide / first time
2. 技术拆解 step by step / breakdown / slow version / form tutorial
3. 常见错误修正 how to fix / mistakes / why am I not improving
4. 特定人群变体 for women / for kids / for seniors / over 40 / youth
5. 目标导向 for weight loss / fat burning / muscle gain / cardio
6. 时间挑战 / 结果故事 30 day challenge / I tried X / transformation / results
7. 明星 / 高阶参照 具体运动员/艺人名字 + tutorial / breakdown
8. 设备 / 场地 at home / no equipment / with dumbbells / in gym
9. 对比 / 决策 vs / which is better / difference between / should I
10. 数量化 / 工具化 how many / how long / calories / times per week / plan

④ 生成方式

本次 query 由 LLM（Claude）根据维度名 + 上述10类角度框架批量生成，非人工逐条编写。隐式生成约束：难度梯度均匀（beginner/intermediate各占）、意图多样（how-to/comparison/challenge/gear）、人群变体 ≥20%、宽泛词+长尾词均有。

⑤ 已知局限（后续可改）

问题当前状态改进方向
条数与缺口强度脱钩手动分三档按 gap_score 动态分配：n = 10 + floor(score/500)×5
重复/相似 query未去重embedding 相似度去重，cosine > 0.92 合并
搜索量未验证纯 LLM 生成接入 Google Trends / SerpAPI 过滤低热度词
维度匹配准确率未验证生成后用分类器回判，过滤跑偏的 query

步骤8：Hermit 种子下发

每条 query 下发 YT + TK 两个平台（IG 无搜索 label，不下发）：

seed_name = gap_{l2_abbrev}_{md5[:6]}_{platform} (max 60 chars) origin = sports_gap_v5_{platform}_{date} # Track A = sports_viral_{platform}_{date} # Track B config = {"keyword": query, "limit": 50} type = one_shot 并发：ThreadPoolExecutor(max_workers=10)
